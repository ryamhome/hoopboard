# HoopBoard 可優化可能性清單

## A. 球場規格引擎
- FIBA / NBA / NCAA / 半場切換。
- 所有線條使用實際尺寸換算，不再目測硬畫。
- 加入「檢查」按鈕，可查看目前場地參數。

## B. 場地風格系統
- 經典木地板、NBA暗色風、紫金風、藍金風、街頭水泥。
- 不使用官方 NBA / 球隊 Logo，降低商標風險。
- 後續可加入自訂配色、地板貼圖、中心Logo上傳。

## C. 戰術動畫
- 目前可用多幀 keyframe。
- 後續可加入每個球員自己的時間軸、暫停點、傳球時間點。
- 可新增戰術分類：進攻、區域聯防、全場壓迫、邊線球、底線球。

## D. 框選與拖曳
- 建議維持選取工具中可框選。
- 後續加入球員鎖定、群組命名、群組複製。

## E. 儲存與分享
- 目前有 Firebase 與本機儲存方向。
- 後續可拆 JSON 戰術檔，讓每個戰術可匯入/匯出。

## F. 使用者體驗
- 右側增加「目標使用者評價」：教練、球員、設計者三種角度。
- 加入新手導覽：先放球員 → 新增幀 → 移動球員 → 播放。

## G. 技術安全
- Firebase 金鑰公開是前端常態，但必須靠 Firebase Rules 限制讀寫權限。
- 建議後續加入 host / guest 權限分級。

## H. 架構升級
- 第一階段：index.html + style.css + js/main.js。
- 第二階段：animation.js / state.js / render.js / input.js / court-engine.js。
- 第三階段：戰術資料庫、主題資料庫、測試檔分離。
